# Learner Profile

## Podstawowe informacje

**Cel:** Przygotowanie do pracy jako DevOps/Infrastructure Engineer
**Styl nauki:** Praktyczny — nauka przez robienie, nie czytanie
**Dostępność:** Regularne sesje, fokus na ciągłości

## Aktualny poziom (startowy)

| Technologia | Poziom startowy | Uwagi |
|-------------|-----------------|-------|
| Terraform   | junior          | Zna podstawy HCL, nie pisał modułów |
| Bash        | junior          | Zna podstawy, set -euo pipefail, pętle |
| Ansible     | junior          | Widział playbooki, nie pisał ról |
| Kubernetes  | brak            | Do wprowadzenia po Terraform |
| Docker      | brak            | Do wprowadzenia po Bash |
| CI/CD       | brak            | Do wprowadzenia po Ansible |

## Mocne strony

- Rozumie koncepcje sieciowe (VPC, subnets, routing)
- Zna podstawy Linuksa
- Potrafi czytać dokumentację techniczną

## Obszary do rozwinięcia

- Pisanie kodu produkcyjnego (nie tylko ćwiczeniowego)
- Myślenie o bezpieczeństwie (secrety, uprawnienia)
- Uzasadnianie decyzji technicznych (interview readiness)

## Preferencje

- Preferuje zadania w stylu work-ticket (realne scenariusze)
- Zadania powinny budować jeden spójny projekt, nie być oderwane
- Ważne: wskazówki "jak powiedzieć o tym na rozmowie"
