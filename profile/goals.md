# Cele nauki

## Cel główny

Uzyskanie zatrudnienia jako DevOps Engineer / Infrastructure Engineer w ciągu 6–12 miesięcy.

## Cele szczegółowe

### Terraform
- [ ] Pisać infrastrukturę jako kod od zera
- [ ] Organizować kod w moduły wielokrotnego użytku
- [ ] Zarządzać stanem (state, backend, workspaces)
- [ ] Zarządzać środowiskami dev/staging/prod

### Bash
- [ ] Pisać skrypty administracyjne i automatyzacyjne
- [ ] Przetwarzać logi i dane
- [ ] Obsługiwać błędy i exit codes poprawnie
- [ ] Pisać skrypty nadające się do cron i CI

### Ansible
- [ ] Pisać playbooki wdrożeniowe
- [ ] Organizować kod w role (role structure)
- [ ] Używać Vault do zarządzania sekretami
- [ ] Integrować Ansible z CI/CD

### Gotowość do pracy

- [ ] Rozwiązywać bugfixy w istniejącym kodzie
- [ ] Robić code review i wskazywać problemy
- [ ] Reagować na incydenty operacyjne
- [ ] Uzasadniać decyzje techniczne na rozmowie

## KPI

- Minimum 2 sesje tygodniowo
- Wynik > 7/10 w 80% sesji
- Ukończyć ścieżkę projektową dla każdej technologii
