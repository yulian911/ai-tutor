# System oceniania — Scorecard

## Osie oceny (0–10 każda)

| Oś | Co sprawdzać |
|----|--------------|
| **Poprawność** | Czy spełnia wszystkie wymagania z `task.md`? |
| **Czytelność** | Nazewnictwo, struktura, formatowanie |
| **Best practices** | Standardy specyficzne dla technologii |
| **Bezpieczeństwo** | Hardcoded secrets, uprawnienia, attack surface |
| **Gotowość do pracy** | Czy można to wdrożyć w prawdziwej firmie? |
| **Samodzielność** | Czy użytkownik przemyślał rozwiązanie, czy kopiował szkielet? |

**Wynik końcowy** = średnia arytmetyczna z 6 osi.

## Progi

| Wynik | Decyzja |
|-------|---------|
| ≥ 7.0 | Zaliczone — można przejść dalej |
| < 7.0 | Niezaliczone — zaplanuj powtórkę tematu |
| < 5.0 | Wymagana natychmiastowa powtórka (nie awansuj) |

## Awans poziomu

Awans wymaga **3 kolejnych sesji z wynikiem ≥ 8** na tym samym poziomie.  
Nie skaczemy poziomów (np. `junior` → `mid-ready` jest niedozwolone).

Ścieżka: `junior` → `junior+` → `mid-ready` → `project-capstone`

## Failure tags

Jeśli wynik w dowolnej osi < 7, zapisz temat jako `failure_tag` w `progress/spaced-repetition.md`.  
Wróć do niego po 2–3 sesjach. Nie uznawaj tematu za zamknięty po jednym wykonaniu.
